# CLAUDE.md

## Writing style

- Never use em-dashes (the long dash character, U+2014) in user-visible text: headings, paragraphs, button labels, form copy, marketing copy, JSX text content, alt text, markdown documents, commit messages, and PR descriptions. Use a comma, a colon, parentheses, or split the sentence in two.
- Code comments are exempt. Em-dashes are fine inside `//` and `/* */` comments.
- When editing a user-visible string that already contains an em-dash, rewrite it.
